# Blender操作ガイド（このプロジェクト用）

Mixamoのモーションを`sample.vrm`（VRoidキャラ）にリターゲットしてVRMA(`.vrma`)として書き出す作業を、Blenderを触ったことがない前提でまとめたもの。

---

## 1. 視点操作（まずこれだけ覚えれば動ける）

| やりたいこと | 操作 |
|---|---|
| 視点を回転 | マウス中央ボタン（ホイール）を押しながらドラッグ |
| 視点を平行移動（パン） | Shift + 中央ボタン ドラッグ |
| ズームイン/アウト | マウスホイールを回す |
| 正面から見る | テンキー **1** |
| 背面から見る | Ctrl + テンキー **1** |
| 右側面から見る | テンキー **3** |
| 真上から見る | テンキー **7** |
| 選択中のものにズームして合わせる | テンキー **.**（ピリオド） |
| 全体を画面に収める | **Home**キー |

テンキーがないキーボードの場合は、上部メニューの「ビュー」からも同じ視点に切り替えられる。

### 表示モード（重さに直結）
右上のシェーディングアイコン（球が4つ並んでる場所）で切り替え：
- **ソリッド**（灰色の球）: 一番軽い。骨や動きを確認するときはこれ
- **マテリアルプレビュー**（縞模様の球）: 質感付きで見れるが、髪の半透明描画のせいでVRoidキャラに近づくと重くなりがち。見た目確認のときだけ使う
- ショートカットは **Z** キーでパイメニューからも選べる

## 2. 選択・アウトライナー

- 画面右上の**アウトライナー**パネルに、シーン内の全オブジェクト（Armature/Body/Face/Hairなど）がツリー表示される
- クリックで選択。選択中のオブジェクトはオレンジ色の輪郭で表示される
- 目のアイコンで表示/非表示、カメラアイコンでレンダリング時の表示/非表示を切り替えられる

---

## 3. タイムライン・アニメーション再生

画面下部の帯がタイムライン。

| やりたいこと | 操作 |
|---|---|
| 好きなフレームに飛ぶ | タイムライン上でクリック、または再生ヘッド（縦線）をドラッグ |
| 1フレームずつ送る | タイムラインにマウスを乗せた状態で **←→**キー |
| 再生/停止 | **スペース**キー |
| 最初のフレームに戻る | **Shift + ←** |
| 最後のフレームに飛ぶ | **Shift + →** |

タイムライン右側に「開始」「終了」という数値フィールドがある。これが**この先で一番重要**（4章参照）。

## 4. このプロジェクトのリターゲット手順

Mixamoの`.fbx`を読み込んで`sample.vrm`のキャラに動きを移し、`.vrma`として書き出すまでの一連の流れ。

### 手順
1. **クリーンな状態で始める**: 新規ファイル（`Ctrl+N` → 「全般」）か、`sample.vrm`だけが入った専用ファイルを用意する。使い回すと過去のジェスチャーのゴミ（アーマチュアやアクション）が溜まって事故る
2. **VRoidキャラを読み込む**: `ファイル > インポート > VRM`で`public/avatar/sample.vrm`を選択
3. **Mixamoのfbxを読み込む**: `ファイル > インポート > FBX`で対象の`.fbx`（例: `tools/X Bot@Walk.fbx`）を選択。`mixamorig:Hips`などのボーンを持つ新しいArmatureが追加される
4. **人型ボーンを自動割り当て**: 読み込んだMixamoのArmatureを選択した状態で、右のサイドバー（Nキーで開閉）の「VRM」タブ →「ヒューマノイド」→「自動でヒューマノイドボーンを割り当て」を押す。`mixamorig:Hips → hips`のような対応が自動で入る（うまく入らないボーンだけ手動で直せばOK）
5. **★ここが今回のバグの元凶: フレーム範囲を実際のアクションの長さに合わせる**
   - タイムライン下部の「開始」「終了」を見る。新規ファイルはデフォルトで**終了=250**になっている
   - 読み込んだアクションの本当の長さは、Mixamoアーマチュアを選んだ状態でドープシート／アクションエディタを開くと確認できる（大体1〜10秒程度のはず）
   - **終了フレームがアクションの実際の最後のフレームと一致していないと、書き出したVRMAが「足りない」か「最後のポーズで固まったまま無駄に長い」状態になる**（今回、腕組み・drinkは短すぎて途中で打ち切られ、walkは長すぎて最後のポーズで7秒以上固まっていた）
6. **VRMAとして書き出す**: `ファイル > エクスポート > VRM Animation`。書き出し対象にMixamoのArmature（人型ボーンを割り当てた方）を指定して保存
7. アプリ側（`public/avatar/*.vrma`）に配置して、Playgroundで実際に再生して目視確認する

### やらかしがちなポイント
- **frame_endを直さずに書き出す**（4-5番を飛ばす）→ 今回の全バグの原因
- **同じファイルを使い回して前のジェスチャーのアーマチュアが残ったまま作業する** → 名前がArmature.001, .002...と増えていき、どれがどのアクションか分からなくなる
- **ベイクしたアクションをリネームせず放置** → 次のリターゲットで上書きされて中身が消える。名前を変えて盾アイコン（フェイクユーザー）を有効にしておくと保護できる

### ★重要: 「Blenderで骨が正しく動いてる」＝「アプリでも正しく動く」ではない

Blenderのタイムラインで骨(Armature)がちゃんと動いて見えても、それだけでは書き出し後のVRMAが正しい保証にはならない。理由は3つ：

1. **ヒューマノイドボーンの対応付けミス**: 自動割り当てで左右が入れ替わっていたり、変なボーンに割り当たっていたりすると、元の骨の動き自体は正常でもVRMに変換した瞬間に壊れる
2. **Restポーズ(T-pose)のズレ**: 書き出しは「アーマチュアの基準姿勢からの差分」で計算されるので、基準姿勢自体がズレていると動きが歪む
3. **frame_end**（上記参照）: Blenderのタイムライン再生も「終了」フィールドの範囲でしか再生・ループしないため、通しで再生して最後が自然にループするかまで見れば、ある程度ここは検知できる

なので確認は必ず2段階でセットにする: **(1) Blenderで骨の動きを確認 → (2) 書き出してアプリ（ブラウザ）で実際に再生して確認**。(1)だけで「直った」と判断しない。

## 5. ショートカット早見表

| キー | 効果 |
|---|---|
| `Ctrl + N` | 新規ファイル |
| `Ctrl + S` | 上書き保存 |
| `Ctrl + Shift + S` | 名前を付けて保存 |
| `N` | 3Dビューのサイドバー表示/非表示（VRMタブもここ） |
| `Tab` | オブジェクトモード ⇔ 編集/ポーズモード切り替え（ボーン編集時によく使う） |
| `Z` | シェーディングモード切り替えパイメニュー |
| `A` | 全選択 / `Alt+A` | 全選択解除 |
| `H` | 選択物を非表示 / `Alt+H` | 全て再表示 |

## 6. 用語メモ

- **Armature（アーマチュア）**: 骨組み（スケルトン）のオブジェクト。VRoidキャラ本体にも、Mixamoのfbxを読み込んだ側にもそれぞれ存在する
- **アクション（Action）**: 1つのアニメーションデータのまとまり。1つのArmatureが複数のアクションを持てる（切り替えて使う）
- **ポーズモード**: ボーンを動かす専用モード。`Tab`キーでオブジェクトモードから切り替え
- **ヒューマノイドボーン**: VRM規格で決まっている「hips/spine/leftUpperArm」などの標準ボーン名。これに正しく対応付けないとVRMA書き出しができない
