# MIRAGE — 要件定義書

> 作成: 2026-06-28 / 仮プロジェクト名: mirage（空中に浮かぶ幻のスマホ）

---

## 1. コンセプト

### 「デバイスのいらない、未来のスマホ」

来場者がモニターの前で手を動かすと、空中にスマホUIが浮かび、
**実機を一切触らずに指（ピンチ）で操作できる。**

- 手 = 入力デバイス（実機ゼロ）
- 一言で伝わる: 「SF映画のアレ」
- 説明不要。触った瞬間にわかる。

これが今までの没案（宇宙スカイダイビング等）と違う点:
**ビジュアルじゃなくコンセプトが先にあり、「で、何？」が無い。**

## 3. 方式（確定）

### マジックミラー方式（PC + Webカメラ）

```
来場者（実空間で手を動かす）
   ↓ Webカメラが撮る
モニター = 鏡（自分の姿 + 空中スマホUIが映る）
   ↓
鏡を見ながら手で操作
```

- 観客全員が同じモニターを見られる = 展示=見世物として成立
- カメラ映像は**左右反転（ミラー）**して出す → 手の動きが直感的になる
- **フォールバック**: どうしても「実際の手元に出したい」ならMeta Quest 3（パススルー + ハンドトラッキング）。ただし本人1人しか見られない

## 4. 技術スタック

- **Vite + React + TypeScript**（クライアント完結・軽量）
- **React Three Fiber + drei**（3DのスマホUI）
- **@mediapipe/tasks-vision**（HandLandmarker = 手の21点トラッキング）
- wasm + モデルは `public/mediapipe/` にローカル同梱（**オフラインでも動く** = 展示会場のネット不安を回避）

### 操作の中核
- 深度（指が画面に触れたか）は単眼カメラだと不安定
  → **ピンチ（親指先4 と 人差し指先8 の距離）= タップ** で回避。これは安定して取れる
- 指先ホバー → ピンチで決定、の2段階

## 6. リスクと対策

| リスク | 対策 |
|--------|------|
| 単眼カメラで深度が曖昧 | ピンチ操作で回避 |
| 照明で手検出が不安定 | 展示位置の照明を事前テスト / numHands・閾値調整 |
| パフォーマンス（カメラ解析 + 3D） | GPU delegate / 解像度調整 / 粒子数制御 |
| 会場のネット不安 | wasm・モデルをローカル同梱（対応済み） |
| 長時間稼働での落ち | Phase 4 で安定性テスト |

---

## 7. 「完成」の定義（展示可能ライン）

1. ✅ 空中にスマホUIが出て、ピンチで操作できる
2. ✅ 初見の来場者が説明なしで触れる
3. ✅ 触った瞬間に「すごい」と言わせる反応・演出
4. ✅ 数時間の連続稼働で落ちない
5. ✅ 会場のネット環境に依存しない
