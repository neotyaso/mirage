## 手順

### STEP 1. スマホをWebカメラにする
- **DroidCam** か **Iriun**（無料・iPhone/Android）をスマホとWindows PCに入れる
- できれば **USBケーブル接続**（WiFiより低遅延 = 反応が気持ちいい）

### STEP 2. dev server起動
```
cd C:\プログラミング\mirage
pnpm dev
```
→ ブラウザで **http://localhost:5173/**（使用中なら 5174 など。ターミナル表示のURLを見る）

### STEP 3. Phase 0の動作確認（検証）
- カメラ許可を押す
- 手をかざす → 関節に**青い点21個**
- 親指と人差し指をくっつける → **緑の円**が出れば成功
- チェックする観点:
  - [ ] 検出の精度（指がちゃんと追えるか）
  - [ ] 遅延（ピンチ→反応の速さ）
  - [ ] 照明を変えた時の安定性
- ⚠️ カメラが切り替わらない / 別のカメラが映る場合 → **Claudeに「カメラ選択UI足して」と頼む**（`getUserMedia`のdeviceId指定を追加する）

---

## STEP 4. Phase 1: 空中スマホUI（R3F）← 明日のメイン作業

検証が通ったらここに着手。タスク順:

1. R3Fの `<Canvas>` を導入し、3Dの「浮かぶスマホ」パネルを表示
2. 手のランドマーク（人差し指先=8）を3D空間にマッピング → **指先カーソル**
3. カーソルがスマホUIのボタンに乗ったら**ホバー表示**
4. **ピンチ（親指4×人差し指8の距離）でタップ確定** → ボタンが光る/押し込む反応
5. スマホ出現アニメ（ポンッと出る）

※ 現状の `App.tsx` はCanvas2Dの検証用ビュー。Phase 1ではR3Fベースに作り替える（検証ロジック=`useHandTracking`フックはそのまま流用できる）。
